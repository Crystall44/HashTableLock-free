# Неблокирующая хеш-таблица, множество, мультимножество

Реализация хеш-таблицы с с lock-free вставкой, автоматическим расширением, а также
структур «множество» (Set) и «мультимножество» (Multiset) на её основе.

# Структура проекта
  
hashtable.h - заголовочный файл хеш-таблицы  
hashtable.c - реализация хеш-таблицы (FNV-1a, lock-free вставка, рехеширование)  
set.h / set.c - множество на основе хеш-таблицы  
multiset.h / multiset.c - мультимножество на основе хеш-таблицы  
main.c - автоматические тесты и примеры использования  
README.md - этот файл

# Особенности

Хеш-функция: FNV-1a (Fowler–Noll–Vo), 32 бита  

Для каждого байта ключа:  
1. hash ^= byte - XOR с байтом  
2. hash *= 16777619 - умножение на простое число

Почему FNV-1a:

FNV-1a была разработана специально для хеш-таблиц и протестирована на реальных данных. Её главные достоинства:

Простота. Вся функция - это цикл из двух операций (XOR и умножение).

Скорость. На каждый байт ключа выполняется всего одно умножение - одна из самых быстрых операций процессора

Хорошая лавинность. Умножение на большое простое число (16777619) «размазывает» биты так, что даже маленькое изменение ключа даёт совершенно другой хеш.

Работа с любыми данными. Функция оперирует сырыми байтами, поэтому неважно, что выступает ключом — строка, число или бинарные данные.

Детерминированность. Одинаковый ключ всегда даёт одинаковый хеш на любой платформе.

# Разрешение коллизий: метод цепочек

Каждая ячейка таблицы хранит связный список элементов с одинаковым хешем.  
При хорошей хеш-функции и адекватном размере таблицы длина цепочек мала.

# Автоматическое расширение (рехеширование)  

Когда длина цепочки превышает порог (по умолчанию 5 элементов), таблица автоматически удваивается, и все элементы перераспределяются.

# Неблокирующая (lock-free) вставка

Для многопоточной безопасности используется  
InterlockedCompareExchangePointer из Windows API - аналог CAS (compare-and-swap).

Алгоритм вставки:  
1. Создаём новый узел  
2. Читаем голову цепочки  
3. Пытаемся атомарно заменить голову через CAS  
4. Если другой поток уже изменил голову - пробуем снова

Почему через Windows API:  
Visual Studio (MSVC) не поддерживает <stdatomic.h> из стандарта C11.  
Используются нативные Windows-функции, которые работают на всех версиях.

# Множество (Set)

Надстройка над хеш-таблицей. Хранит только уникальные элементы.  
value всегда = 1 (маркер присутствия).

Операции: add, remove, contains, union, intersection, difference.

# Мультимножество (Multiset)

Надстройка над хеш-таблицей. Хранит элементы со счётчиком вхождений.  
value = количество.

Операции: add(key, count), remove(key, count), count(key), contains(key).

# Сборка и запуск

Visual Studio:
  
1. Создать проект: File → New → Project → Console App  
2. Удалить автоматически созданный .cpp файл  
3. Добавить файлы с расширением .c:  
- hashtable.c  
- set.c  
- multiset.c  
- main.c  
4. Добавить заголовочные файлы: hashtable.h, set.h, multiset.h  
5. Собрать: Build → Build Solution (Ctrl+Shift+B)  
6. Запустить: Debug → Start Without Debugging (Ctrl+F5)

Командная строка (MinGW / GCC):
 
gcc -Wall -std=c11 main.c hashtable.c set.c multiset.c -o hashtest.exe  
hashtest.exe
